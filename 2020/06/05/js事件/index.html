js事件

事件

事件就是文档或者浏览器窗口中发生一些特定的交互瞬间。

事件流

事件流描述的是从页面中接收事件的顺序
两类：
IE的事件流（事件冒泡）：事件开始是由最具体的元素接收，然后向上传递，知道文档
div–>body–>html–>document
Netscape Communicator的事件流（事件捕获）：不太具体的结点应该更早接收事件，而最具体的元素应该最后接收事件。向下传递。
document–>htm–>body–>div
DOM事件流：
三个阶段：
事件捕获阶段
处于目标阶段
事件冒泡阶段

事件处理程序：

事件处理程序（事件侦听器）就是响应某个事件的函数。
指定和删除事件处理程序
addEventListener()
removeEventListener()
接受3个参数
要处理的事件名、作为事件处理程序的函数、布尔值（值为true是捕获阶段调用，反之在冒泡阶段调用）
IE事件处理程序
有两个类似的方法
attachEvent()
detachEvent()
接受两个参数，IE 是冒泡事件流，所以没有第三参数。事件都添加在事件冒泡阶段
跨浏览器的事件处理程序

事件对象

在触发DOM上的某个事件时，会产生一个对象事件event。
DOM中的事件对象
event
属性：
bubbles 事件是否冒泡
cancelable 是否可以取消事件默认行为
currentTarget 事件当前正在处理的元素 等于this对象
defaultPrevented 是否阻止默认事件 比如a链接跳转，form的提交按钮
detail 与事件相关的信息 只读
eventPhase 调用事件处理阶段
target 事件的目标
trusted 为true表示浏览器生成的，否则由开发人员通过JavaScript创建的。
type 被触发的事件类型
view 与事件相关的抽象视图
方法
preventDefault() 取消默认行为 前提是在cancelable为true时才能使用
stopImmediatePropagation() 取消事件进一步捕获与冒泡，同时阻止任何事件处理程序被调用。
stopPropagation() 取消事件的进一步捕获与冒泡，如果bubbles为true的时候可以使用
IE事件对象
window.event
属性:
cancelBubble 是否取消冒泡事件
returnValue 是否可以取消事件默认行为
srcElement 事件的目标
type 事件类型
跨浏览器对象
兼容浏览器 event||window.event

事件类型：

UI事件 用户与页面元素交互时触发
load
unload
abort 停止下载过程时
error
select
resize 窗口大小变化时
scroll
焦点事件 元素获取或者失去焦点时
鼠标事件 用户通过鼠标在页面上执行操作时
滚轮事件 使用鼠标滚轮时
文本事件 在文档中输入文本时
键盘事件 通过键盘在页面上执行操作时
合成事件 当为IME时，输入字符时
变动事件 底层DOM结构发生变化时

事件委托

对于事件程序过多问题的处理，利用事件冒泡，只指定一个事件处理程序，就可以管理某一类型的所有事件。
优点：
节约内存，减少事件注册
动态绑定事件，对于新增元素可以自动绑定，无需再次绑定。

事件模型：

DOM0级模型（原始事件模型）
    在该模型中，事件不会传播。没有事件流概念。
    可通过html绑定也可通过js绑定
IE事件模型
    共有两种阶段
        事件处理阶段、事件冒泡阶段
DOM2级模型
    属于W3C标准模型，有三个阶段
        事件捕获阶段、事件处理阶段、事件冒泡阶段

事件循环

javascript有一个基于事件循环的并发模型，事件循环负责执行代码、收集和处理事件以及执行对列中的子任务。
js的同步事件放入主线程中，形成执行栈。
js的异步事件放入任务队列中，而异步又分为微任务与宏任务。
宏队列 setTimeout、setInterval、setImmediate、I/O、UI rendering
微任务 promise.then、process.nextTick
执行顺序：
先执行主线程中所有事件
遇到宏任务放入宏队列，遇到微任务放入微队列
再进入任务队列中，执行所有微任务
在执行所有宏任务
重复3和4步骤
