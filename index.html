js-严格模式

语句：

"use strict"

概念：

在严格的条件下运行JavaScript

声明：

在作用域顶端声明（脚本或者函数顶部），整个作用域都处于严格模式状态。

优势：

消除代码不安全之处，保证代码的安全性。
提高编译效率，增加运行速度
为以后JavaScript做好铺垫

要求：

不允许使用未声明的变量
不允许删除变量、对象和函数，以及不能删除的不允许删的属性
不允许变量重名
不允许使用八进制
不允许使用转义字符
不允许对只读属性赋值
不允许使用getter获取的属性进行赋值
变量名不能使用eval\arguments
禁止this指向全局对象
新增一些保留字
    implements interface let package private protected public static yield

js事件

事件

事件就是文档或者浏览器窗口中发生一些特定的交互瞬间。

事件流

事件流描述的是从页面中接收事件的顺序
两类：
IE的事件流（事件冒泡）：事件开始是由最具体的元素接收，然后向上传递，知道文档
div–>body–>html–>document
Netscape Communicator的事件流（事件捕获）：不太具体的结点应该更早接收事件，而最具体的元素应该最后接收事件。向下传递。
document–>htm–>body–>div
DOM事件流：
三个阶段：
事件捕获阶段
处于目标阶段
事件冒泡阶段

事件处理程序：

事件处理程序（事件侦听器）就是响应某个事件的函数。
指定和删除事件处理程序
addEventListener()
removeEventListener()
接受3个参数
要处理的事件名、作为事件处理程序的函数、布尔值（值为true是捕获阶段调用，反之在冒泡阶段调用）
IE事件处理程序
有两个类似的方法
attachEvent()
detachEvent()
接受两个参数，IE 是冒泡事件流，所以没有第三参数。事件都添加在事件冒泡阶段
跨浏览器的事件处理程序

事件对象

在触发DOM上的某个事件时，会产生一个对象事件event。

事件委托

对于事件程序过多问题的处理，利用事件冒泡，只指定一个事件处理程序，就可以管理某一类型的所有事件。
优点：
节约内存，减少事件注册
动态绑定事件，对于新增元素可以自动绑定，无需再次绑定。

事件模型：

DOM0级模型（原始事件模型）
    在该模型中，事件不会传播。没有事件流概念。
    可通过html绑定也可通过js绑定
IE事件模型
    共有两种阶段
        事件处理阶段、事件冒泡阶段
DOM2级模型
    属于W3C标准模型，有三个阶段
        事件捕获阶段、事件处理阶段、事件冒泡阶段

事件循环

javascript有一个基于事件循环的并发模型，事件循环负责执行代码、收集和处理事件以及执行对列中的子任务。
js的同步事件放入主线程中，形成执行栈。
js的异步事件放入任务队列中，而异步又分为微任务与宏任务。
宏队列 setTimeout、setInterval、setImmediate、I/O、UI rendering
微任务 promise.then、process.nextTick
执行顺序：
先执行主线程中所有事件
遇到宏任务放入宏队列，遇到微任务放入微队列
再进入任务队列中，执行所有微任务
在执行所有宏任务
重复3和4步骤

基数排序

是一种用在卡片排序机上的算法。
给定n个d为数，其中每一个数位有k个可能的取值，如果RADIX-SORT使用的稳定排序方法耗时Θ(n+k，那么它就可以在Θ(d(n+k))时间内将这些数排好。
给定一个b位数和任何正整数r<=b,如果RADIX-SORT使用的稳定排序算法对数据存储区间是0到k的输入进行排序耗时Θ(n+k),那么它就可以在Θ((b/r)(n+2^r))时间内将这些数排好。
证明 
    以二进制划分 b为比特长 划分为b/r份 每份数值大小是0-2^r 
    T(n) = O(d(n+k))=>T(n) = O(b/r(n+2^r))
    r = lgn时 函数最小 T(n) = O(bn/lgn)

分治法
思想：将原问题分解成几个规模较小的但类似于原问题的子问题，递归地求解这些子问题，然后在合并这些子问题的解来建立原问题的解。
